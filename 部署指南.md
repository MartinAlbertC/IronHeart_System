# IronHeart System 部署指南

## 一、从 GitHub 获取项目

```bash
git clone https://github.com/MartinAlbertC/IronHeart_System.git
cd IronHeart_System
```

### 下载模型文件

模型文件未包含在 Git 仓库中（超过 GitHub 100MB 限制），使用下载脚本获取：

```bash
python download_models.py
```

该脚本会自动下载以下模型：

| 模型 | 大小 | 用途 | 下载源 |
|------|------|------|--------|
| Florence-2-base | ~890MB | 场景描述 | HuggingFace |
| wespeaker-cnceleb-resnet34-LM | ~26MB | 声纹编码 | HuggingFace |
| yolo12n.pt | ~5.4MB | YOLO 备用模型 | Ultralytics GitHub |
| yolo26n.pt | ~5.3MB | YOLO 目标检测 | Ultralytics GitHub |
| yolo26n-cls.pt | ~5.6MB | YOLO 分类模型 | Ultralytics GitHub |

> 已存在的文件会自动跳过，支持断点续传（删除未完成的文件后重新运行）。
> funasr（SenseVoiceSmall ASR 模型）首次运行时自动下载，无需手动处理。
> 若目标机器无网络，需从本机 `~/.cache/funasr/` 打包一并拷贝。

### 测试数据（可选）

自行准备测试视频放入 `data/` 目录：

```bash
mkdir -p data
# 将视频文件放入 data/
```

## 二、目标机器需要的环境

### 环境总览

| 组件 | 版本要求 | 用途 | 需要 GPU？ |
|------|----------|------|-----------|
| Python（conda 环境） | 3.9 + CUDA | A 层（视觉/音频推理） | **必须** |
| Python（系统） | 3.11+ | B/C/D 层 + Broker | 不需要 |
| Node.js | >= 18 | E 层 | 不需要 |
| PostgreSQL | >= 14 + pgvector 扩展 | C 层记忆存储 | 不需要 |

### 环境 1：GPU Python（A 层专用）

需要 CUDA capable GPU（建议 >= 6GB VRAM）。

```bash
conda create -n d2lGPU python=3.9
conda activate d2lGPU

# PyTorch with CUDA（根据 CUDA 版本选择对应 url）
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118

# A 层核心依赖
pip install ultralytics>=8.0.0        # YOLO 目标检测 + BoTSort 跟踪
pip install insightface>=0.7.3        # 人脸检测 + ArcFace 编码
pip install onnxruntime-gpu>=1.15.0   # wespeaker ONNX 推理
pip install opencv-python>=4.8.0      # 图像处理
pip install modelscope                # Florence-2 模型加载
pip install funasr                    # SenseVoiceSmall 语音识别
pip install numpy>=1.24.0
pip install Pillow>=9.0.0
```

> Silero-VAD 内置在 torch 中，无需额外安装。
> insightface 还依赖 onnxruntime（非 GPU 版也可工作）。

### 环境 2：系统 Python（B/C/D 层 + Broker）

```bash
pip install -r requirements.txt
```

或手动安装：

```bash
pip install sentence-transformers>=2.2.0  # D 层 embedding（bge-small-zh）
pip install psycopg2-binary>=2.9.0        # C 层 PostgreSQL 驱动
pip install psycopg[binary]>=3.1.0
pip install pgvector>=0.2.0               # 向量相似度查询
pip install fastapi>=0.100.0              # C 层 HTTP API
pip install uvicorn>=0.20.0
pip install flask>=3.0.0                  # 前端服务
pip install flask-cors>=4.0.0
pip install numpy>=1.24.0
pip install pydantic>=2.0.0
pip install openai>=4.20.0                # D 层 LLM 调用（DeepSeek 兼容接口）
pip install volcenginesdkarkruntime>=1.0.0 # D 层豆包 API
```

### 环境 3：Node.js（E 层）

```bash
cd e_layer
npm install          # 安装 express, openai, uuid
npx tsc              # 编译 TypeScript → dist/
cd ..
```

### 外部服务：PostgreSQL + pgvector

```bash
# 安装 PostgreSQL 14+（Ubuntu 示例）
sudo apt install postgresql-14 postgresql-server-dev-14

# 安装 pgvector 扩展
git clone --branch v0.5.1 https://github.com/pgvector/pgvector.git
cd pgvector && make && sudo make install

# 创建数据库和用户
sudo -u postgres psql
CREATE EXTENSION vector;
CREATE USER ai_system WITH PASSWORD 'secretpassword';
CREATE DATABASE memory_db OWNER ai_system;
```

默认连接参数（均可通过环境变量覆盖）：

| 参数 | 默认值 | 环境变量 |
|------|--------|----------|
| host | localhost | `PG_HOST` |
| port | 5432 | `PG_PORT` |
| user | ai_system | `PG_USER` |
| password | secretpassword | `PG_PASSWORD` |
| dbname | memory_db | `PG_DBNAME` |

## 三、部署后需要修改的地方

### 1. `start_all.py` 第 34 行 — 硬编码的 conda 路径（必须改）

```python
# 当前（写死了原始机器的用户路径）：
D2LGPU_PYTHON = r"C:\Users\CWQ98\anaconda3\envs\d2lGPU\python.exe"

# 推荐改为：
import os, shutil
D2LGPU_PYTHON = os.environ.get("D2LGPU_PYTHON", shutil.which("python"))
```

### 2. `stop_all.py` — Windows WMIC 命令（Linux/Mac 需改）

当前使用 `wmic process where ...` + `taskkill`，仅适用于 Windows。

Linux/Mac 替代方案：改为 `pkill -f "keyword"` 或 `ps aux | grep keyword | awk '{print $2}' | xargs kill`。

### 3. `config.json` — API 密钥（必须改）

```json
{
    "llm_api_url": "https://api.deepseek.com/chat/completions",
    "llm_api_key": "替换为你的 DeepSeek API Key",
    "doubao": {
        "api_key": "替换为豆包 API Key",
        "endpoint_id": "替换为 endpoint ID",
        "base_url": "https://ark.cn-beijing.volces.com/api/v3"
    }
}
```

### 4. `d_layer/config.py` 第 49 行 — 豆包 API Key（必须改）

```python
DOUBAO_API_KEY = os.getenv("DOUBAO_API_KEY", "替换或通过环境变量传入")
```

### 5. `a_layer/config.py` 第 19 行 — GPU 设备（视情况改）

```python
DEVICE = "cuda"   # 无 GPU 的机器改为 "cpu"（A 层推理会极慢）
```

### 6. PostgreSQL 连接信息（视部署改）

可设置环境变量，也可直接修改 `c_layer/config.py`：

```bash
export PG_HOST=localhost
export PG_PORT=5432
export PG_USER=ai_system
export PG_PASSWORD=secretpassword
export PG_DBNAME=memory_db
```

## 五、快速部署步骤（目标机器）

```bash
# 1. 克隆项目
git clone https://github.com/MartinAlbertC/IronHeart_System.git
cd IronHeart_System

# 2. 下载模型文件
python download_models.py

# 3. 创建运行时目录
mkdir -p logs outputs data

# 4. 配置 PostgreSQL + pgvector（见上文第二节）

# 5. 安装系统 Python 依赖（B/C/D 层 + Broker）
pip install -r requirements.txt

# 6. 配置 GPU conda 环境（A 层）
conda create -n d2lGPU python=3.9
conda activate d2lGPU
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
pip install ultralytics insightface onnxruntime-gpu opencv-python modelscope funasr

# 7. 编译 E 层
cd e_layer && npm install && npx tsc && cd ..

# 8. 修改 start_all.py 中的 GPU Python 路径
# 9. 修改 config.json 中的 API keys
# 10. 放入测试视频到 data/

# 11. 启动系统
conda activate d2lGPU   # 如果 start_all.py 中硬编码了路径则不需要
python start_all.py --video data/test.mp4

# 可选：限制处理帧数（调试用）
python start_all.py --video data/test.mp4 --max-frames 300

# 不需要 A 层（无视频输入）：
python start_all.py --mode no-a
```
